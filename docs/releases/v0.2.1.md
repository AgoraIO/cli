# Agora CLI v0.2.1

Patch release focused on agent/CI automation reliability, quickstart clone hardening, and release-artifact naming cleanup.

## Highlights

- **Headless-safe defaults** — `agora open` no longer auto-launches a browser in CI or non-TTY sessions; `agora upgrade` skips in-place self-update in CI unless explicitly opted in.
- **Deterministic init** — `agora init --yes` now requires `--template` and returns `projectSelectionReason` in JSON for agent branching.
- **Quickstart clone fixes** — git credential helpers disabled for public clones; friendly error codes for missing git, bad refs, and malformed repo overrides; `clone:override` progress when env overrides are active.
- **Release archive rename** — GitHub release assets are now `agora-cli_v*` (was `agora-cli-go_v*` for v0.1.7–v0.2.0).

## Upgrade notes

### Direct installer (`install.sh` / `install.ps1`)

Fresh installs and re-runs of the installer always fetch the latest install script and the correct archive name — no action needed.

If you are on **v0.1.7 through v0.2.0** and `agora upgrade` fails to download v0.2.1+, re-run the installer once:

```bash
curl -fsSL https://raw.githubusercontent.com/AgoraIO/cli/main/install.sh | sh
```

Those older binaries embed upgrade logic that only knows the `agora-cli-go_*` archive prefix. After one reinstall, future upgrades use `agora-cli_*`.

### npm / package managers

Use your package manager as usual (`npm update -g agoraio-cli`, etc.). No archive-name change affects npm installs.

### Automation scripts

| Before | After |
| ------ | ----- |
| `agora init demo --yes` (silent template pick) | `agora init demo --template nextjs --yes` (required) |
| `agora upgrade` in CI jobs | `agora upgrade --check --json` (non-mutating) |
| `agora open --target docs` in CI | URL-only by default; pass `--browser` to force launch |

Set `AGORA_ALLOW_UPGRADE_IN_CI=1` only when a CI job intentionally needs to mutate the installed binary.

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/AgoraIO/cli/main/install.sh | sh
agora --version
```

Or pin this release:

```bash
curl -fsSL https://raw.githubusercontent.com/AgoraIO/cli/main/install.sh | sh -s -- --version 0.2.1
```
